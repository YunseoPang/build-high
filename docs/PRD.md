## **# 🚀 Build-High 제품 요구사항 정의서 (PRD)**

### **## 1. 프로젝트 개요**

- **서비스 명칭:** Build-High (빌드하이)
- **핵심 가치:** 엔지니어를 위한 스터디 및 프로젝트 팀 빌딩 매칭 플랫폼
- **기술 스택:** Next.js (App Router), Supabase, Tailwind CSS, v0.dev
- **타겟 유저:** 대학생 개발자, 취준생, 사이드 프로젝트 희망자
- **디자인 컨셉:** v0 기반의 보라색 포인트를 활용한 전문적이고 깔끔한 UI

---

### **## 2. 주요 기능 및 요구사항**

### **### 2.1 메인 대시보드 및 리스트**

- **통계 대시보드:** 활성 멤버 수, 진행 중인 프로젝트, 매칭 성공률 데이터를 시각화하여 노출합니다.
- **카테고리 필터링:** '개발', '스터디', '사이드 프로젝트' 탭을 통해 모집글을 분류하여 조회합니다.
- **정렬 로직:** 기본적으로 최신 등록순(`created_at` 내림차순)으로 정렬합니다.

### **### 2.2 모집글 관리 (CRUD)**

- **생성(Create):** 카테고리, 제목(필수), 요약(필수), 상세 내용, 태그(기술/요일/장소/기간), 연락처 링크를 포함하는 폼을 제공합니다.
- **조회(Read):** 카드 형태의 리스트 뷰와 게시글 상세 페이지를 제공합니다.
- **삭제(Delete):** 작성자 본인만 게시글을 삭제할 수 있도록 권한을 제한합니다 (User UUID 검증).

---

### **## 3. 데이터 아키텍처 (DB 스키마)**

### **### Table: `studies`**

- `id`: uuid (Primary Key) - 게시글 고유 식별자
- `title`: varchar(100), NOT NULL - 게시글 제목
- `category`: text, NOT NULL - 카테고리 (Development, Study, Project 중 선택)
- `summary`: text, NOT NULL - 리스트 카드 노출용 짧은 설명
- `content`: text - 상세 페이지용 본문 내용
- `tags`: jsonb - 기술 스택 및 환경 정보 태그 배열
- `contact`: text, NOT NULL - 오픈카톡 등 외부 연락처 링크
- `created_at`: timestamptz, Default: now() - 생성 일시

---

### **## 4. 유저 플로우 (User Flow)**

1. **랜딩(Landing):** 플랫폼의 전체 통계와 모집 배너를 확인합니다.
2. **탐색(Discovery):** 카테고리 필터를 활용해 최신 모집글 리스트를 브라우징합니다.
3. **작성(Creation):** '새 글 작성' 메뉴를 통해 프로젝트 정보를 등록합니다 (로그인 안내 및 진입 레이어 구현).
4. **연결(Connection):** 상세 페이지에서 제공되는 연락처 링크를 통해 매칭을 시작합니다.

---

### **## 5. 엔지니어링 제약 및 품질 관리 (QA)**

- **빈 상태(Empty State):** 검색 결과나 리스트가 없을 경우 "첫 번째 프로젝트의 주인공이 되어보세요!"라는 안내 메시지를 출력합니다.
- **유효성 검사:** 제목은 최소 5자 이상이어야 하며, 필수 항목이 누락된 경우 등록을 차단합니다.
- **보안:** Supabase의 RLS(Row Level Security)를 설정하여 데이터 무단 수정을 방지합니다.
